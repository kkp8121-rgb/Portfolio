## 6. 핵심 역량

### 6.1 수직 슬라이스 구현

프론트 화면만 구현하는 방식이 아니라, 기능이 실제 운영까지 도달하는 데 필요한 API, 모바일 브릿지, 결제 검증, 푸시, 운영 콘솔까지 연결한다.

대표 사례:
- DM 실시간 채팅: 프론트 SignalR 훅, 백엔드 Hub, 음성 녹음, 모바일 권한, 읽음 표시까지 연결
- SEO 전환: 백엔드 SEO API, 프론트 SSR 메타데이터, sitemap / robots / hreflang, Search Console까지 연결
- 결제: 앱 IAP, 백엔드 검증, Google Play / App Store 상품 설정, 구독 갱신 알림까지 연결

### 6.2 운영 중 시스템의 안전한 전환

기능을 새로 만드는 것보다 어려운 운영 중 변경을 여러 번 수행했다.

- LG U+ 아바타 빌더: Addressables → AssetBundle 시스템 1주일 전환
- BowwowDefence: ChargeCount 기반 스킬 사용 모델 → CoolDown 기반 모델 재설계
- Talkain / StoryBeginz: CSR 중심 페이지 → SEO용 SSR 메타/본문 점진 전환
- 모바일 앱: WebView 크래시 복구, 네이티브 키보드 모드 동적 전환

### 6.3 복잡한 상태와 엣지케이스 처리

사용자 시나리오가 많고 상태가 얽힌 영역을 오래 다뤘다.

- 아바타 파츠 결합: 모자, 헤어, 수염, 맨몸, 컬러피커, BlendShape 충돌 처리
- 콘텐츠 플랫폼: 사용자/캐릭터/채널/PD 프로필, NSFW, 차단, 비공개, 다국어, 결제 권한 처리
- 모바일 WebView: OAuth, 결제, 키보드, 노치, 스크롤 락, 백그라운드 복귀 처리

### 6.4 운영 콘솔까지 포함한 제품 감각

개발 코드만 다루지 않고 실제 출시·수익화·검색 노출·분석·인증까지 담당했다.

- Google AdSense / AdMob / Google Ad Manager / IMA SDK / House Ad
- Google Play Console / App Store Connect / Apple Developer Portal
- Google Search Console / sitemap / robots / hreflang
- GCP Pub/Sub / Google Play RTDN / StoreKit / Stripe
- OneSignal / Firebase Analytics / GA4 / AppsFlyer / Facebook SDK

## 7. 대표 사례 5건

### 7.1 SEO / SSR 안전 전환 + 백엔드 SEO 인프라 신설

**상황**: Talkain / StoryBeginz는 `'use client'` 중심 구조로 인해 봇이 본문을 충분히 인덱싱하지 못하는 문제가 있었다.  
**행동**: 7가지 안티패턴을 정리한 자체 가이드를 만들고, 백엔드에 `SeoService` 및 StoryBeginz 전용 컨트롤러를 신설했다. 프론트에서는 Feed / Reels에 SSR 메타데이터와 본문을 추가하고 인터랙션은 CSR로 유지했다.  
**결과**: 핵심 페이지 SEO 인프라를 1주일 내 전환했고, 부수적으로 production 콘솔에 JWT가 노출되는 P0 보안 이슈를 발견·차단했다.

관련 문서:
- [avatarplay-frontend.md](./avatarplay-frontend.md)
- [avatarplay-chat-api.md](./avatarplay-chat-api.md)

### 7.2 Android Native Module로 WebView 키보드 UX 해결

**상황**: DM, 채팅, 댓글 화면마다 키보드 동작 요구가 달라 React Native 레벨만으로는 일관된 해결이 어려웠다.  
**행동**: `KeyboardModeModule`을 Java/Kotlin 기반 Android Native Module로 직접 작성하고, 화면별 `windowSoftInputMode` 동적 토글을 WebView 브릿지에 연결했다.  
**결과**: Expo Managed Workflow의 제약을 우회하면서 화면별 키보드 UX를 분리했고, StoryBeginz 0.0.43까지 안정 빌드 운영에 반영했다.

관련 문서:
- [avatarplay-app.md](./avatarplay-app.md)

### 7.3 LG U+ 아바타 빌더의 파츠 결합 안정화

**상황**: 3D 아바타 빌더는 헤어, 얼굴, 의상, 악세서리, 신발, 모자, 수염, 색상, BlendShape가 동시에 얽히는 구조였다.  
**행동**: `ObjectManager_Avatar.cs`, Face Editor, Color Picker, AssetBundle Manager를 중심으로 파츠 교체와 저장/로드 엣지케이스를 정리했다.  
**결과**: 8월 한 달 84커밋으로 모자/수염/맨몸/컬러피커 충돌, 저장 직후 맨몸 로딩, 단색 배경 캐릭터 사라짐 등 사용자 시나리오 기반 결함을 집중 해결했다.

관련 문서:
- [uplus-avatar-client.md](./uplus-avatar-client.md)

### 7.4 BowwowDefence 스킬/발사체 코어 구축

**상황**: 2D 디펜스 게임에서 다양한 스킬과 발사체를 빠르게 추가할 수 있는 구조가 필요했다.  
**행동**: `SkillManager`, `SkillBase`, `Projectile`, `Missile_Chain`, `Missile_Pierce`, `Fire_Splash`, `SkillModuleDamageAround` 등으로 모듈형 스킬/발사체 구조를 만들었다.  
**결과**: 단일/연쇄/관통/광역/부채꼴/사후효과 발사체를 공통 구조로 처리하고, ChargeCount → CoolDown 전환과 데이터/UI 마이그레이션까지 수행했다.

관련 문서:
- [BowwowDefence.md](./BowwowDefence.md)

### 7.5 운영 콘솔과 수익화 인프라 직접 처리

**상황**: 콘텐츠 플랫폼은 코드 구현 외에도 광고, 앱 출시, 검색 노출, 구독 상품, 결제 알림, 인증, 분석 도구가 모두 필요했다.  
**행동**: AdSense 승인, AdMob, GAM/IMA, Play Console, App Store Connect, GCP Pub/Sub, OneSignal, Search Console, Stripe, 본인인증, NSFW 정책까지 직접 다뤘다.  
**결과**: 개발 기능이 실제 매출·출시·검색·푸시·결제 검증으로 이어지는 전체 운영 경로를 혼자 연결할 수 있는 역량을 확보했다.
