# Talkain / StoryBeginz — 백엔드 API

> .NET 8 / C# 기반 백엔드 API. 웹 클라이언트와 모바일 앱에 데이터·실시간 메시징·결제 검증·SEO 페이지를 제공한다. **프론트엔드 메인 개발자가 백엔드까지 직접 손대는 1인 풀스택 협업 패턴**의 백엔드 파트.

- **재직 형태**: 자체 서비스 (현직)
- **활동 기간**: 2024년 11월 ~ 진행 중 (약 18개월)
- **역할**: 백엔드 보조 컨트리뷰터 — 본인 프론트 작업과 결합되는 영역 직접 수정

## 3. 담당 영역

### 3.1 SEO 시스템 신설 (2026 Q2 주력)
프론트엔드 SSR 전환과 동시에 백엔드에 SEO 전용 서비스 / 컨트롤러 신설.

- `SeoService` 신규 — SEO 메타·본문 생성 로직 일원화
- StoryBeginz 전용 컨트롤러 신설 (메인 페이지 / Explore / SEO)
- Feed / Reels SEO 전용 엔드포인트
- 다국어 hreflang 지원

> 프론트엔드 SSR 전환이 백엔드 변경 라운드트립 없이 진행됨. **한 명이 양쪽을 다 만지는 구조의 장점**.

### 3.2 포스트 & 커뮤니티
가장 많이 작업한 영역. 신규 엔드포인트와 정책 변경 반영.

- 공유 / 조회 / 피드 조회 신규 API
- 포스트 정보 응답 확장 (`isFollow`, `commentCount`, `dislike`, 언어 타입)
- 리포스트 카운트 정책, 원본 삭제 시 추천 제외
- 비공개 포스트 / 콘텐츠 결격사유(NSFW · 삭제 · 비활성 계정) 필터링
- 공모전 랜덤 추출 API, 한줄 요약 + 번역

### 3.3 프로필 시스템
- 활성 프로필 기준 비공개 항목 **서버 사이드 필터링** 도입
- 캐릭터 채널 URL 링크 키 관리
- PD 정보 갱신·회원가입 버그 해결
- 댓글 프로필 타입 분류

### 3.4 결제 시스템 (IAP)
- Google IAP 검증, 구독 갱신 알림 수신
- Apple IAP 검증
- 결제 엣지케이스 — 본인 콘텐츠 유료 결제 시 스타 차감 방지 등
- iOS IAP 단품 구매 버그 해결

### 3.5 실시간 채팅 (SignalR Hub)
- DM Hub 메시지 프로토콜 안정화
- DM ID 발급, 마이크(음성 녹음) 연동
- 친구(맞팔) 제한 정책 변경 대응

### 3.6 데이터·인프라
- 패킷 / DTO 정의 (프론트와 스키마 동기화)
- 데이터 액세스 레이어
- 멀티 테넌트 서비스 ID 분기

### 3.7 보안·필터링
- NSFW 필터 강화 / 해제 정책
- 차단된 콘텐츠 / 비활성 계정 비노출
- 외부 서비스 권한 예외 처리

### 3.8 친구 / 팔로우
- 팔로우 → 언팔 → 팔로우 시 중복 데이터 발생하던 동시성 버그 해결
- 친구 북마크 PD 프로필 통일, URL 링크 키 발급·중복 검사 개선

### 3.9 피드 & Explore
- 인기 대화 캐릭터 챗봇 우선 노출
- 피드 소스 episode / content 우선 매핑
- 릴스 피드 visibility / NSFW / 차단 필터
- 콘텐츠 비디오 URL prefix 누락 수정

### 3.10 알림 (OneSignal)
- OneSignal 키 Rotate 무중단 대응
- 채팅방 목록 레드닷(읽지 않음) 정렬

### 3.11 환경 설정 (멀티 테넌트)
Development / Staging / StoryBeginzDev / 기본 환경별 `appsettings.json` 분리 관리.

## 대표 사례. 백엔드 SEO 인프라 신설로 프론트 SSR 전환 가속화

**Situation** — 프론트엔드를 SSR로 전환하려면 백엔드가 SEO 메타데이터·본문을 공급해야 했음. 기존 컨트롤러를 그대로 활용하면 사이드이펙트 위험.

**Task** — 운영 중인 API를 건드리지 않고 SEO 응답을 분리해서 제공하는 경로 설계.

**Action**
- `SeoService.cs` 신규 작성 — SEO 메타·본문 생성을 한 곳으로 일원화
- 브랜드 전용 컨트롤러(`StoryBeginzMainPageController` / `StoryBeginzExploreController`) 신설
- Feed / Reels용 SEO 전용 엔드포인트 추가
- 다국어 hreflang 지원

**Result**
- 프론트엔드 SSR 전환이 백엔드 변경 라운드트립 없이 한 사람 손에서 동시 진행됨
- 기존 API에 사이드이펙트 0건
- 핵심 페이지 SEO 인덱싱 기반 완성

## 협업 특징

- **프론트엔드 메인 + 백엔드 보조**라는 1인 풀스택 협업 패턴
- 다른 백엔드 메인 컨트리뷰터들과 영역 분담
- 본인 프론트 작업에 필요한 **수직 슬라이스**를 백엔드까지 직접 처리
- 단기 우회보다 산업 표준 본질안(스키마 표준화, 정책 일원화 등)을 선호하는 의사결정
