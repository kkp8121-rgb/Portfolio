# Talkain / StoryBeginz — 웹 프론트엔드

> Next.js 15 기반 종합 콘텐츠 플랫폼의 메인 웹 클라이언트. 영상·웹툰·소설·캐릭터 챗(AI)·쇼츠·커뮤니티 포스트·공모전을 한 코드베이스에서 다루며, 환경변수 분기로 **Talkain**과 **StoryBeginz** 두 브랜드를 동시 운영한다.

- **재직 형태**: 자체 서비스 (현직)
- **활동 기간**: 2024년 9월 ~ 진행 중 (약 20개월)
- **역할**: **프론트엔드 메인 컨트리뷰터**

## 4. 담당 도메인 (12개 영역)

### 4.1 콘텐츠 생성·뷰어
시리즈 / 에피소드 / 싱글 / 웹툰 / 영상 통합 CRUD. 작가 도구(에피소드 분기 트리거, 대화 템플릿, 순서 변경, 초안 저장), Shaka Player 기반 영상 뷰어, 멀티 포맷 미디어 업로드(iOS 크래시 대응 포함).

### 4.2 홈피드 & Reels (영상 숏폼)
플랫폼의 핵심 노출 영역. 무한 스크롤·자동 재생·일시 mute, 광고 자연 삽입, 노치 대응, 핀치 줌 방지, 콘텐츠 가시성·NSFW·차단 필터.

### 4.3 프로필 시스템
다층 프로필(일반 / 캐릭터 / 채널 / PD) 통합 컴포넌트. 활성 프로필 전환 시 탭/권한 자동 갱신, 비공개 항목 서버 필터링, 팔로우·친구·차단·추천·북마크.

### 4.4 채팅 시스템
AI 캐릭터 챗 + DM 1:1 실시간 채팅. SignalR 클라이언트 연동, DM 음성 녹음, 채팅방 읽음 표시 정렬, OS별 키보드/스크롤 분기 처리.

### 4.5 결제 & 구독
구독 플랜 UI, 결제 내역, 지갑·카드 등록. 본인 콘텐츠 유료 결제 시 스타 차감 방지 등 결제 엣지케이스.

### 4.6 게이미피케이션
상점, 가상 재화(스타), 시간 보상, 데일리 리워드, 구독 등급별 혜택.

### 4.7 SEO & SSR/CSR 최적화 (2026 Q2 주력)
운영 중인 SPA에서 SSR 메타데이터·본문을 안전하게 추가하는 점진적 전환. 자체 가이드 문서화([대표 사례](#대표-사례-1-운영-중-서비스의-ssrcsr-안전-전환--seo-인프라-구축)).

### 4.8 보안 강화
P0 보안 패치(JWT 콘솔 노출 차단), XSS 방지(`innerHTML` 제거), 401 인터셉터 + 자동 토큰 갱신, NSFW 필터, 신고 시스템.

### 4.9 다국어(i18n)
5개 언어(ko/en/ja/zh-CN/zh-TW), URL 경로 기반 라우팅, `(&)` 접두어 미번역 자동 탐지 시스템, Google Sheet 자동 동기화 파이프라인.

### 4.10 광고 시스템
AdSense 중복 push 방지, IMA SDK 영상 광고 자연 삽입, House Ad 자체 컴포넌트. AdSense for Video 2026-05-31 종료 대응 검토.

### 4.11 WebView 브릿지 (앱-웹 통신)
모바일 앱과의 통합 진입점. 결제·푸시·키보드·노치·스크롤 락·OAuth 콜백·권한 요청을 통합 프로토콜로 처리.

### 4.12 검색
통합 검색, 채팅 내 검색.

## 대표 사례 1. 운영 중 서비스의 SSR/CSR 안전 전환 + SEO 인프라 구축

**Situation** — 운영 중인 SPA에서 `'use client'` 남발로 SEO 약점. 봇이 본문을 인덱싱하지 못해 검색 노출이 저조했음.

**Task** — 한 번에 `'use client'`를 떼면 권한 미스매치 / 하이드레이션 / 토큰 캐시 오염 등 다발성 버그 위험. 안전 전환 경로 필요.

**Action**
1. **자체 가이드 문서화**: `'use client'` 제거 시 발생 가능한 **7가지 안티패턴**(이중 호출 · 권한 미스매치 · 하이드레이션 · 토큰 캐시 오염 · 환경변수 · 브라우저 API · ISR stale) 정리 → 사내 스킬로 정형화
2. **점진적 전환**: 인터랙션 무거운 페이지(캐릭터·콘텐츠 상세)는 SSR fetch 제거하고 CSR-only로 유지, 메타데이터만 SSR로
3. **본문 SSR 추가**: Feed / Reels는 SSR로 본문·메타만 렌더링하고 인터랙션은 클라이언트 유지
4. **백엔드 연계**: 백엔드에 SEO 전용 서비스/컨트롤러를 신설하고 다국어 hreflang 지원 (프론트·백 동시 작업)
5. **안전성 점검**: 잘못된 id 차단, 공개 필터, 프로필 페이지 SSR fetch 제거

**Result**
- 1주일 내 핵심 페이지에 SSR 메타데이터·본문 적용 완료
- 부수효과로 **production 콘솔에 JWT가 노출되던 P0 보안 이슈 발견·차단**
- 이후 다른 페이지에서 동일 패턴 작업 시 재현 가능한 가이드로 활용

## 대표 사례 2. 실시간 DM 채팅 풀스택 구현

**Situation** — AI 캐릭터 챗 외에 사용자 간 1:1 실시간 DM 요구. 음성 녹음, 읽음 표시, 친구(맞팔) 제한 등 복합 요구사항.

**Task** — 프론트 클라이언트, .NET SignalR Hub, 알림(OneSignal) 동기화, 모바일 WebView 환경 호환을 **혼자 풀스택**으로 처리.

**Action**
- 백엔드 SignalR Hub 메시지 프로토콜 설계·안정화
- 프론트 SignalR 훅으로 연결 / 재연결 / 토큰 갱신 통합
- DM 음성 녹음 — 모바일 마이크 권한을 앱·웹 양쪽에서 처리
- 채팅방 리스트 읽음 표시(레드닷) 정렬 버그 해소

**Result** — 약 2주 만에 DM 1차 완성. 이후 점진적 안정화 진행. 친구 제한·비공개 정책 등 정책 변경에도 단일 개발자로 빠르게 대응.
