## 3. 담당 도메인

### 3.1 스킬 시스템 — 가장 깊이 작업한 영역

발사체 6종을 **모듈형으로 설계**해서 신규 스킬 추가 비용을 최소화.

| 발사체 | 메커니즘 |
|--------|---------|
| 직선형 미사일 | 기본 단일 타격 |
| 연쇄형 미사일 | 적 간 연쇄 + 연사 |
| 관통형 미사일 | 관통 횟수에 따른 데미지 감산 |
| 화염 스플래시 | 광역 폭발 |
| 부채꼴 발사체 | 다중 발사 |
| 사후 효과 객체 | 잔류·폭발 |

추가 작업:
- 광역 데미지 모듈에 **타원형 영역** 추가 (원형 한정 → 타원으로 확장)
- 데미지 공식 정형화
- 스킬 자동 장착 로직 + 정렬
- **스킬 사용 모델 재설계** — 충전 카운트 방식에서 쿨다운 기반으로 전환 (데이터 마이그레이션 포함)
- 발사체 잔류 버그 수정 (객체 풀 라이프사이클)

### 3.2 데이터 관리 시스템
ScriptableObject 기반 3계층 데이터 레이어.

- 정적 게임 데이터(스킬/유닛 테이블)
- 유저 진행도·재화·인벤토리
- 장착 아이템 스키마
- 데이터 테이블 인덱스 검증

### 3.3 보물상자 가챠 시스템
Spine 애니메이션 동기화 + 스킵 가능 시퀀스.

- 보물상자 연출 → 카드 등장 → 결과 카드 → 보상 적용 파이프라인
- 재화 타입 보상 추가
- 스킵 시 발생하던 NRE / 잔여 연출 버그를 안전장치로 차단

### 3.4 유닛 & 스킬 보드 UI
- 보드 패널 매니저 (유닛 / 스킬 / 축복 탭 컨트롤)
- 유닛 슬롯 해금·탈부착·자동 장착·자동 정렬
- 슬롯 / 팝업 / 슬라이더 / Fill 색상 시스템
- 하단 탭 네비게이션

### 3.5 축복(Bless) 시스템
- 축복 아이템 + 레벨업 팝업
- 경험치 바, 레벨업 보너스 수치 적용

### 3.6 전투 핵심 로직
- 전투 규칙 / 페이즈 / 승패 판정
- 아군 유닛 / 적 캐릭터 베이스 클래스
- 게임 매니저 상태 머신

### 3.7 Addressables 도입 및 빌드 환경 세팅
대용량 2D 아트 / Spine / 프리팹을 비동기 로드.

## 4. 기술적 특이점

### 4.1 모듈형 스킬·발사체 구조
**Strategy + Decorator 패턴**으로 발사체를 모듈화. 베이스 발사체 + 행동 모듈(Chain / Pierce / Splash / Fan) 조합으로 6종을 처리. 신규 발사체 추가 시 기존 코드를 건드리지 않고 모듈만 조립 가능.

### 4.2 스킬 사용 모델 재설계 (운영 중 마이그레이션)
초기 충전식 스킬 사용 모델을 운영 도중 쿨다운 기반으로 전환. 데이터 마이그레이션 + UI(슬라이더 / 색상) + 자동 사용 로직을 동시 수정해서 깨지지 않게 처리.

### 4.3 ScriptableObject 기반 데이터 레이어
정적 게임 데이터를 ScriptableObject로 관리하여 **디자이너가 코드 빌드 없이 밸런스를 조정**할 수 있게 함. 정적/동적 데이터를 3계층으로 분리.

### 4.4 가챠 연출 파이프라인
보물상자 → 카드 등장 → 결과 카드 → 보상 적용까지의 Spine 애니메이션 동기화. 스킵 시 발생하던 NRE / 잔여 연출 버그를 안전장치(Off Block)로 차단.

## 5. 게임 시스템 다이어그램

```
[GameManager] ─ 상태/페이즈 관리
   ├─ [BattleRule] ─ 승패·페이즈 판정
   ├─ [BoardPanelManager] ─ 인게임 보드 UI 전환
   │     ├─ 유닛 패널 → 슬롯·팝업·자동장착
   │     └─ 스킬 패널 → 슬롯·팝업·자동장착
   ├─ [SkillManager] ─ 스킬 사용·쿨다운·자동장착
   │     └─ SkillBase
   │          └─ SkillModule(광역 데미지 등)
   │                └─ Projectile
   │                     ├─ Missile / Chain / Pierce
   │                     ├─ Splash
   │                     └─ ActionObject / Explosion (사후 효과)
   ├─ [UnitItemManager] ─ 유닛 인벤토리·장착
   └─ [DataManager + LocalDataTable + UserDataManager]
         └─ EquipItemData

[가챠] PopupTreasureBox
   └─ TreasureCard → TreasureResultCard → SummonReward
```
